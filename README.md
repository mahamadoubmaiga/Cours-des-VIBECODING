## 💡 La Philosophie du Vibe Coding

**Le vibe coding** signifie :
- Se concentrer sur ce que l'application doit faire, pas comment la coder parfaitement
- Laisser les outils modernes faire le gros du travail
- Apprendre en construisant, pas en mémorisant
- Publier un travail imparfait et l'améliorer

**Ce N'EST PAS :**
- De l'enseignement traditionnel du code
- Mémoriser la syntaxe
- Construire du code parfait du premier coup
- Attendre d'être « prêt » pour publier

---

## 🛠️ La Stack (Tout Gratuit pour Commencer)

1. **Firebase Studio** — Backend, auth, base de données et hébergement en un seul endroit
2. **GitHub** — Ta source de vérité et l'historique de ton projet
3. **Cursor** — Éditeur de code propulsé par l'IA qui comprend ton projet

Tu n'as pas besoin de 300 €/mois en outils. Tu as besoin de ces trois.

### Le Workflow

**Voici comment tu les utiliseras vraiment :**

1. **Construire les ossatures du projet dans Firebase Studio** — Utilise des invites pour créer la structure de base
2. **Publier depuis Firebase Studio** — Crée ton projet dans Firebase Console
3. **Activer les services dans Firebase Console** — Active Auth, Firestore, Hébergement (facilite tout plus tard)
4. **Synchroniser avec GitHub** — Firebase Studio crée ton dépôt GitHub automatiquement (supprime la friction de configuration GitHub)
5. **Cloner dans Cursor** — Construire vite avec l'IA (tous les services déjà configurés)
6. **Synchroniser de nouveau et déployer** — Déployer depuis Firebase Studio (déploiement en un clic)

**Construire les ossatures → Publier → Activer les services → Synchroniser avec GitHub → Construire dans Cursor → Déployer. Répéter.**

**Points clés :**
- Construire les ossatures du projet d'abord (la structure)
- Publier avant de synchroniser avec GitHub (active les services)
- Activer tous les services tôt (facilite le travail dans Cursor)
- Firebase Studio crée ton dépôt GitHub automatiquement
- Garde les dépôts privés et ne commit pas de clés secrètes

Chaque outil fait ce qu'il fait de mieux. Tu ne te bats pas contre les outils.
